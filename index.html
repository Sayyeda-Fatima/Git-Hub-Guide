An easy guide to Git

Introduction

This is a beginer friendly guide to Git.

What is Git?

Git is a Distributed Version Control System. In addition to being distributed, Git has been designed with performance, security and flexibility in mind.
In this tutorial we will see some important Git commands and concepts that every beginer should know.

Let's begin then: First of all you will need to download and install Git on your computer. Once you have installed git now you can create local repisitories and start your version control.

Importance of Git:

Git is very important for programmers ass it allows them to save changes in every version and keep a histrory of these changes in case they want to return to previous version when error occurs. Also it allows user to collebrate with other people as all people can have same code and will save changes to same repository, so, anyone can see these changes.

Repository:

The first and most common concept of Git is repositories. All the file versions are saved in a repository. In simple words it is a folder that stores all the version and history of changes in our code.

Creating a Git Repository

To create a repository, you can press right click on mouse, press new from the menu and create a folder and open that folder in Command line terminal. Then we use following command to make a git repository in required folder :

-git init-


After creating the folder open it and again press right click on mouse. Choose "Open in terminal" from menu. It will open a command line prompt in your folder. Now you can write " git init " command and initialize a git repository in that folder.





Now we have succesfully initialized a git repository in our folder. When we intialize a git repository in a folder a " .git " folder is created in our original folder and that .git folder is our repository but is hidden mostly.

Staging area:

Staging area is a temporary place where every file that we want to add to our repository must stay before commiting it to repository. It lets git know what changes in the file are going to occur. We can add our files to git staging area by using command:

-git add-

We can use git add command with '.' to add all files to staging area or we can give name of the file to add to staging area.

or

To see if our files are added to staging area or not, or even to see status or condition of our repository at anytime we use -git status- command.

Commiting Files to Repository

Here we need to remember that -git add- only adds our files to staging area and not to our repository directly. To add our files from staging area to repository, we use following command:

-git commit -m ""-

Where -m is used to add a appropriate message to our commit.

Now if we use -git status- it will show us that our reposoitory is up to date and there are no new or modified files in our staging area to be commited.

As we can see on -git status- it tells us that there is noting to commit.
