An easy guide to Git

Introduction

This is a beginer friendly guide to Git.

What is Git?

Git is a Distributed Version Control System. In addition to being distributed, Git has been designed with performance, security and flexibility in mind.
In this tutorial we will see some important Git commands and concepts that every beginer should know.

Let's begin then: First of all you will need to download and install Git on your computer. Once you have installed git now you can create local repisitories and start your version control.

Importance of Git:

Git is very important for programmers ass it allows them to save changes in every version and keep a histrory of these changes in case they want to return to previous version when error occurs. Also it allows user to collebrate with other people as all people can have same code and will save changes to same repository, so, anyone can see these changes.

Repository:

The first and most common concept of Git is repositories. All the file versions are saved in a repository. In simple words it is a folder that stores all the version and history of changes in our code.

Creating a Git Repository

To create a repository, you can press right click on mouse, press new from the menu and create a folder and open that folder in Command line terminal. Then we use following command to make a git repository in required folder :

-git init-


After creating the folder open it and again press right click on mouse. Choose "Open in terminal" from menu. It will open a command line prompt in your folder. Now you can write " git init " command and initialize a git repository in that folder.





Now we have succesfully initialized a git repository in our folder. When we intialize a git repository in a folder a " .git " folder is created in our original folder and that .git folder is our repository but is hidden mostly.

Staging area:

Staging area is a temporary place where every file that we want to add to our repository must stay before commiting it to repository. It lets git know what changes in the file are going to occur. We can add our files to git staging area by using command:

-git add-

We can use git add command with '.' to add all files to staging area or we can give name of the file to add to staging area.

or

To see if our files are added to staging area or not, or even to see status or condition of our repository at anytime we use -git status- command.

Here the green files show that they are currently in staging area. If we modify a file it will show in red which means that we need to add those files to staging area e.g:

In the above example you can see that when we change in file1.txt it shows in red and have a status of modified. Now we have to again -git add- to add to staging area.

Commiting Files to Repository

Here we need to remember that -git add- only adds our files to staging area and not to our repository directly. To add our files from staging area to repository, we use following command:

-git commit -m ""-

Where -m is used to add a appropriate message to our commit.

Now if we use -git status- it will show us that our reposoitory is up to date and there are no new or modified files in our staging area to be commited.

As we can see on -git status- it tells us that there is noting to commit.

Branches:

Git branches are effectively a pointer to a snapshot of your changes. When you want to add a new feature or fix a bug —no matter how big or how small— you create a new branch to encapsulate your changes so that if that new feautre or code create some fatal error your original code is still safe in your main branch.
You can use following command to see branches in your repository:

-git branch-

The star symbol in the start of branch name shows that it the current working branch, means all the changes will be stored to this branch. Branch master is made by default when we initiate our repository and work as our main branch.

Creating a new branch:

To create a new branch we use following command:

-git checkout -b name-

where name is the name of new branch.

Here we use -git checkout- command to move to new branch and -b attribute is used to create new branch and then move to it.

As you can see we have created new branches and are now currently at branch3.
Important Note: All these branches will have same code, as master branch, on creation.

Git Hub

Now that we have learned some of the basic it we can now learn about Git Hub, a platform for version control.
Git Hub is a online platform that helps programmers to manage repositories online.

How to create a Git Hub account
How to create a repository on Git Hub

The repositories on Git Hub are refferd to as remote repositories. We can push data from our local repository on our device to our remote repository on Git Hub.

How to Push to Remote Reposoitory

To push data to remote repository we first need to link our local repository to our remote repository on Git Hub. To do this we use following command:

-git remote add origin url-

where url is the url link of your repository.

Now to push files to our remote repository we use command:

-git push -u origin main-

where main is name of master branch.

We use main as Git Hub do not allow to push masrter branch.

Now your local repository is saved to your Git Hub repository.

Notice that you have to push to your remote repository every time you commit so your cahnges will be saved to local repository too.



And this is how programmers can keep record of code progress and collaborate with each other easily using git and git hub.

Overview:

  1. git init : To initialize a git repository in your folder.
  2. git add . : To add files to staging area.
  3. git commit -m "" : To save files in local repository.
  4. git branch : To see all the branches in our repository.
  5. git checkout -b : To create a new branch in our repository.
  6. git checkout : To move from one branch to another.
  7. git remote add origin : To link our local repository to remote repository on Git Hub.
  8. git push -u origin : To push files from local repository to remote repository on Git Hub.
